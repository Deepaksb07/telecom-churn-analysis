##  Project Overview
This project analyzes a dataset of **7,043 telecom customers** to identify key indicators of churn. By implementing a hybrid workflow using **SQL for feature engineering** and **Python for visualization**, this analysis highlights high-risk segments and suggests actionable retention strategies.

**Key Goal:** Reduce customer attrition by identifying early warning signs in contract types, tenure, and payment methods.

##  Key Insights & Results
1.  **The "Early Exit" Problem:**
    * **38%** of all churned users leave within the first **3 months**.
    * *Strategic Recommendation:* Implement a "First 90 Days" onboarding campaign to increase engagement during this critical window.

2.  **Contract Vulnerability:**
    * Customers on **Month-to-Month contracts** paying via **Electronic Check** have the highest churn rate (~45%).
    * *Strategic Recommendation:* Offer incentives (e.g., small discounts or data bonuses) for users to switch to 1-year contracts or automatic bank transfers.

3.  **The "Bundle" Effect:**
    * Customers with **1 or fewer add-on services** (like Tech Support or Online Security) are **2x more likely to churn** than those with 3+ services.
    * *Strategic Recommendation:* Push "Bundle Offers" at checkout to make customers "stickier" to the ecosystem.
